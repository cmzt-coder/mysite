    Claude Code 命令大全¶

    一、命令行命令大全¶

    根据 https://code.claude.com/docs/en/cli-reference 页面，以下是 Claude Code 命令行接口（CLI）的所有命令和标志的详细解释。

    这些是你的系统终端（如 CMD, PowerShell, Bash）中直接运行 claude 时可以使用的参数，主要用于控制 Claude Code 的启动方式、行为模式和自动化脚本集成。

    1. 核心命令 (Commands)¶

    这些是启动 Claude Code 的基本方式。

    命令格式 作用详细解释 示例
    claude 启动交互式 REPL。进入默认的对话模式，你可以像在网页版一样与 Claude 聊天，但它能操作你的本地文件。 claude
    claude "query" 带问题启动。启动交互模式，并直接将你双引号里的内容作为第一句话发给 Claude。 claude "explain this project"
    claude -p "query" 单次执行模式 (Print Mode)。非交互式。执行完指令（比如解释代码、修改文件）后直接打印结果并退出。非常适合写进 Shell 脚本。 claude -p "explain main.py"
    claude -c 继续上次会话 (Continue)。自动加载当前目录下最近的一次对话历史，接着聊。 claude -c
    claude -r "<id>" 恢复指定会话 (Resume)。通过 ID 或名称恢复特定的历史会话。 claude -r "auth-task" "完成它"
    claude update 自我更新。将 Claude Code 升级到最新版本。 claude update
    claude mcp 配置 MCP。进入 MCP 服务器配置流程，用于连接外部工具或数据源。 claude mcp

    2. 核心 CLI 标志 (Flags)¶

    这些参数用于自定义 Claude Code 的启动行为，可以配合上述命令使用。

    A. 模式与控制¶

    • --print / -p
    • 作用：打印模式。执行完任务后立即退出，不进入交互界面。通常用于自动化任务或管道操作（Pipe）。
    • 示例：cat logs.txt | claude -p "find errors"
    • --continue / -c
    • 作用：继续会话。加载当前目录下最近的上下文。
    • --resume / -r
    • 作用：指定恢复。恢复特定的会话 ID 或名称。如果后面不跟 ID，会弹出一个交互式列表让你选。
    • --init
    • 作用：初始化启动。运行初始化钩子（Hooks）并启动交互模式。
    • --init-only
    • 作用：仅初始化。运行初始化钩子后直接退出，不进入对话。
    • --verbose
    • 作用：详细日志。显示详细的调试信息，包括完整的对话轮次和工具调用细节，Debug 必备。
    • --version / -v
    • 作用：显示当前版本号。

    B. 权限与安全¶

    • --dangerously-skip-permissions
    • 作用：危险模式。跳过所有的“是否允许执行命令/读写文件”的确认提示。慎用，除非你在完全可控的沙箱环境中。
    • --allow-dangerously-skip-permissions
    • 作用：允许跳过权限。开启此选项后，允许你在会话中通过指令激活“跳过权限”的功能（配合 --permission-mode 使用）。
    • --permission-mode
    • 作用：指定权限模式。比如以 plan 模式启动（只生成计划不执行）。
    • 示例：claude --permission-mode plan

    C. 模型与 Agent 配置¶

    • --model
    • 作用：指定模型。强制使用特定的模型版本（如 sonnet, opus，或具体版本号）。
    • 示例：claude --model claude-3-opus-20240229
    • --agent
    • 作用：指定 Agent。指定当前会话使用某个预设的 Agent 角色。
    • --agents
    • 作用：定义子 Agent。传入一个 JSON 字符串来动态定义子 Agent（包含描述、Prompt、工具）。
    • 示例：claude --agents '{"reviewer": {"prompt": "你是代码审查员..."}}'

    D. 系统提示词 (System Prompt) 定制¶

    • --system-prompt
    • 作用：完全替换默认的系统提示词。你拥有对 Claude 行为的完全控制权，它不再默认知道自己是 Claude Code。
    • --system-prompt-file
    • 作用：同上，但从文件加载提示词内容（仅限 -p 打印模式）。
    • --append-system-prompt
    • 作用：追加提示词。在默认提示词后面加上你的指令（推荐使用，保留了 Claude Code 的原有能力）。
    • 示例：claude --append-system-prompt "所有回复必须用中文"
    • --append-system-prompt-file
    • 作用：同上，从文件追加（仅限 -p 打印模式）。

    E. 输入输出与集成¶

    • --output-format
    • 作用：输出格式。指定输出是纯文本 (text)、JSON (json) 还是流式 JSON (stream-json)。自动化脚本必用。
    • 示例：claude -p "生成配置" --output-format json
    • --input-format
    • 作用：输入格式。指定输入给 Claude 的内容格式。
    • --json-schema
    • 作用：结构化输出。强制 Claude 返回符合指定 JSON Schema 的 JSON 数据（仅限 -p 模式）。
    • --add-dir
    • 作用：添加工作目录。让 Claude 能访问除当前目录以外的其他文件夹。
    • 示例：claude --add-dir ../lib ../utils
    • --chrome / --no-chrome
    • 作用：浏览器集成。开启或禁用 Chrome 浏览器控制能力（用于网页自动化测试）。

    F. 其他高级选项¶

    • --settings：加载额外的设置文件（JSON）。
    • --mcp-config：从指定文件加载 MCP 服务器配置。
    • --strict-mcp-config：严格模式，只加载 --mcp-config 指定的 MCP，忽略其他配置。
    • --plugin-dir：加载指定目录下的插件。
    • --remote：在 claude.ai 网页端创建一个新会话。
    • --teleport：将网页端的会话“传送”回本地终端。
    • --max-budget-usd：设置预算上限。当 API 花费达到指定美元金额时自动停止（仅限 -p 模式）。
    • --max-turns：限制对话轮数。防止死循环，达到指定轮数后强制退出（仅限 -p 模式）。

    总结：最常用的组合¶

    1. 日常开发：直接敲 claude。
    2. 接着昨天干：claude -c。
    3. 写脚本自动化：cat error.log | claude -p "分析错误原因" --output-format json。
    4. 调试问题：claude --verbose。

    二、交互式命令大全¶

    根据 https://code.claude.com/docs/en/interactive-mode 页面内容，Claude Code 的交互式模式（Interactive Mode）提供了一系列命令来管理会话、配置工具和执行任务。

    这些命令通常以 / 开头（内置命令）或以其他符号（如 !、@）作为前缀。以下是所有命令的详细中文解释：

    1. 核心内置命令 (Built-in Commands)¶

    这些是在 Claude Code 对话框中直接输入 / 即可调用的功能指令：

    命令 作用详细解释
    /clear 清屏并清除历史。清除终端屏幕上的显示内容，并重置当前的上下文历史（相当于开始新对话，但保留项目记忆）。
    /compact [instructions] 压缩上下文。当对话过长时，使用此命令将之前的对话历史压缩成摘要，以释放 Token 空间。你可以在后面加上指令（如 /compact 保留关于数据库的讨论）来指定压缩时的重点。
    /config 打开配置面板。启动一个交互式的设置界面，你可以在这里修改主题、默认模型、权限设置等。
    /context 查看上下文使用情况。以彩色网格的形式可视化当前会话占用了多少上下文窗口（Token），帮你判断是否需要压缩。
    /cost 查看费用统计。显示当前会话或账户的 Token 使用量及预估费用（针对付费计划用户）。
    /doctor 健康检查。自动诊断 Claude Code 的安装环境，检查网络连接、权限、依赖项等是否正常。
    /exit 退出程序。结束当前的 Claude Code 会话并关闭程序（也可以使用 Ctrl+D）。
    /export [filename] 导出对话。将当前的对话记录导出保存为文件，或者导出到剪贴板。
    /help 获取帮助。显示使用指南、快捷键列表和可用命令概览。
    /init 初始化项目。在当前目录下生成 CLAUDE.md 文件。这是项目的“记忆文件”，用于存储项目特定的指令、架构规范和常用知识。
    /mcp 管理 MCP 工具。进入 MCP（Model Context Protocol）管理界面，用于连接本地数据库、添加自定义工具或处理 OAuth 认证。
    /memory 编辑记忆文件。直接打开并编辑 CLAUDE.md 文件，让你能快速更新项目的核心上下文。
    /model 切换模型。允许你选择或更改当前使用的 AI 模型（如从 Sonnet 切换到 Opus）。
    /permissions 管理权限。查看或更新 Claude Code 对文件读写、终端执行等操作的权限设置。
    /plan 进入计划模式。直接进入“Plan Mode”，让 Claude 生成详细的任务执行计划而不立即执行代码。
    /rename <name> 重命名会话。给当前会话起个名字，方便以后查找或恢复。
    /resume [session] 恢复会话。通过 ID 或名称恢复之前的某个历史会话，或者打开会话选择器。
    /rewind 回溯/撤销。将对话和代码状态“倒带”回之前的某个时间点（撤销 AI 最近的操作）。也可以双击 Esc 触发。
    /stats 查看统计数据。显示你的每日使用情况、会话历史、连续使用天数以及模型偏好等数据。
    /status 查看状态。打开状态面板，显示当前版本号、登录账户、模型信息和网络连接状态。
    /statusline 设置状态栏。配置 Claude Code 底部状态栏的显示样式和内容。
    /copy 复制回复。将 AI 上一次的回复内容直接复制到系统剪贴板。
    /tasks 管理后台任务。列出并管理当前正在后台运行的任务（如正在跑的测试或服务器进程）。
    /teleport 传送会话。仅限订阅用户，允许你从 claude.ai 网页版恢复并接管一个远程会话。
    /theme 更改主题。修改界面的颜色主题（如切换深色/浅色模式）。
    /todos 查看待办事项。列出当前项目识别到的 TODO 列表或任务进度。
    /usage 查看用量限制。显示当前计划的速率限制（Rate Limit）状态和使用额度（仅限订阅用户）。
    /vim 启用 Vim 模式。开启 Vim 风格的按键绑定，适合习惯 Vim 操作的开发者。

    2. 快捷指令前缀 (Quick Commands)¶

    除了以 / 开头的命令，还有两个特殊的符号用于快速操作：

    • ! (Bash 模式)

    • 作用：直接运行 Shell 命令，不经过 Claude 的自然语言处理。

    • 用法：!npm test 或 !git status。

    • 特点：命令的输出会直接进入对话上下文，AI 可以看到运行结果。支持 Ctrl+B 将长任务转入后台运行。

    • @ (文件引用)

    • 作用：触发文件路径自动补全，显式告诉 Claude 关注某个文件。

    • 用法：输入 @ 后跟随文件名，例如 @main.py。
    • 特点：这会将该文件的内容明确添加到上下文中，确保 AI 读取的是最新版本。

    3. 常用控制快捷键 (General Controls)¶

    虽然不是文本命令，但这些组合键在交互模式下非常重要：

    • Ctrl + C：取消当前的输入或打断 AI 的生成。
    • Ctrl + D：退出会话（EOF）。
    • Ctrl + R：搜索历史命令（Reverse search）。像终端一样搜索你之前输入过的 Prompt。
    • Esc + Esc (双击)：回溯 (Rewind)。快速撤销最近的一轮对话或操作。
    • Ctrl + B：转后台运行。当一个命令（如 npm run dev）正在运行时，按下此键将其转入后台，让你能继续与 Claude 对话。
    • Ctrl + K / Ctrl + U：删除到行尾 / 删除整行（类似 Readline/Bash 快捷键）。
