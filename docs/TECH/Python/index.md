## 内容编写规范

本分类下的所有知识点文档均遵循统一格式，新增内容时请严格参照以下规则，以保证可读性与可扩展性：

### 条目格式

每个知识点为一个 `###` 三级标题，标题格式：

```
### [编号] 知识点名称（yyyy-mm-dd）
```

### 条目字段

每个条目按顺序包含以下固定字段，以 `**字段名**：` 开头，后接内容：

| 字段 | 必填 | 说明 |
|------|------|------|
| **分类** | 是 | 所属大类（语法 / 标准库 / 设计模式 / 性能 / 工具链 等） |
| **标签** | 是 | 逗号分隔的关键词，方便站内搜索 |
| **要点** | 是 | 1-3 句话概括核心结论，帮助快速回忆 |
| **示例** | 否 | 最小可运行代码片段（仅保留关键行），使用围栏代码块并标注语言 `python` |
| **详见** | 否 | 外部参考链接或站内关联文章 |

### 扩展规则

- 编号从 `001` 起递增，与创建日期无关，便于排序和引用
- 新知识追加在对应文件末尾（基础追加到 `01-`，进阶追加到 `02-`）
- 目录区需同步更新条目列表
