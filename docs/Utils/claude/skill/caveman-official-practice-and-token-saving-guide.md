# Caveman 插件使用指南

本指南专为你在 Windows 环境下成功安装的 **`caveman`** 插件及组件量身定制。通过该插件，你可以让 Claude Code 等终端智能体（Agent）彻底脱去人类社交的“废话包袱”，切换至信息密度极高的\*\*“技术野人”**语件，在大幅提升交互延迟（Latency）的同时，为你**疯狂节省约 30% - 80% 的输出（Output）与输入（Input）Token 账单\*\*。

***

## 📌 第一部分：核心指令与压缩等级调节

进入 `claude` 交互界面后，你可以随时通过输入以下控制台指令（或直接用中文命令）来动态调节 AI 的说话“脱水程度”：

| 控制台指令             | 对应中文命令  | 运作表现与原理                                              | 预期节省 Output Token | 最佳适用场景                                     |
| :---------------- | :------ | :--------------------------------------------------- | :---------------- | :----------------------------------------- |
| `/caveman lite`   | 开启轻量模式  | 仅修剪多余的口头禅和客套话，保留基本的自然语言句子结构与冠词。                      | **\~30%**         | 用于生成代码评审（PR Comments）或需要向团队汇报的文档。          |
| `/caveman full`   | 开启完全模式  | **（系统默认等级）** 彻底丢弃多余的连词、修饰词。直奔技术干货，使用符号（如 `->`）与短语表达。 | **\~65%**         | 日常的绝大多数 Agent 自动编码、复杂 Debug 与架构拆解任务。       |
| `/caveman ultra`  | 开启极限模式  | 几乎是电报式的极简主义，满篇只有符号、缩写和碎片化词组。                         | **\~75% - 80%**   | 极度熟悉项目后、多 Agent 协作或内部子智能体（Sub-agent）互相通信时。 |
| `/caveman wenyan` | 开启文言文模式 | 极其硬核的压缩。强迫 AI 使用言简意赅的**中国古代文言文**进行技术总结与调用链阐述。        | **物理级极限最高**       | 追求单次响应延迟（Latency）低到极致的工程调测。                |
| `normal mode`     | 恢复正常模式  | **关闭野人模式**。AI 将瞬间恢复充满礼貌、语法通顺且带有完整解释的标准人类自然语言。        | **0%**            | 当遇到极度复杂的业务逻辑，野人语导致你理解困难时使用。                |

> 💡 **中文化支持说明**：`caveman` 压缩的是“语言的结构和表达风气”，而不是将语言强行翻译。当你用中文与 Claude Code 对话时，它同样会以高度精简的\*\*“中文野人语”\*\*（除文言文模式外）回答你。同时，**代码块、核心函数名、路径和错误日志会 100% 保持精准的字节级原样输出**，绝不乱动。

## 🧰 第二部分：硬核专属高级专项工具箱

除了在聊天中省 Token，你刚才勾选的 `caveman` 子组件还附带了三个贯穿软件开发全生命周期的专属高级命令。请直接在 Claude 聊天框中像使用快捷键一样调用它们：

### 1. 🌟 历史文档极限瘦身：`/caveman-compress <file>`

这是面对大型开源项目时**最推荐优先使用的神技**。在长周期的代码拆解中，项目里肥胖的 `CLAUDE.md`、`README.md` 或 `ARCHITECTURE.md` 会作为 Input 在每一次对话中被 AI 重复读取，造成极大的上下文浪费。

- **使用方法**：在聊天框输入 `/caveman-compress CLAUDE.md`
- **工程效果**：它会直接调用重写算法，将这些大体积的自然语言说明书用高密度的“野人语”重新覆盖改写。**在 100% 完好保留技术实体、URLs、架构和代码块的前提下，直接帮你砍掉约 46% 的长周期 Input Token 预算！**（注：原文件会自动在本地保留一个 `.original.md` 的备份供人类查阅）。

### 2. 一行流代码评审：`/caveman-review`

当你在本地进行完二次开发，想要让 Agent 帮你做代码评审时使用。

- **工程效果**：它绝不会长篇大论，而是提取你的 Git Diff，直接生成一针见血、高密度的一行流（One-liner）硬核批注。
- *示例输出*：`L42: 🔴 bug: user null. Add guard.`

### 3. 极简代码提交：`/caveman-commit`

- **使用方法**：在本地完成代码修改并 `git add` 暂存后，在聊天框输入该指令。
- **工程效果**：Agent 会全自动提取你本地暂存区的修改差异，为你生成高度规范、小于 50 字符、且完全符合 Conventional Commits 规范的紧凑 Git Commit 提交信息。

## 🔒 第三部分：安全刹车机制 (Auto-Clarity)

无需担心 AI 变成“野人”后会因为乱删文件而导致项目崩溃。`caveman` 插件内部集成了\*\*全自动安全越狱（Auto-Clarity）\*\*熔断逻辑。

一旦 Agent 侦测到其即将执行的行为涉及以下敏感或破坏性场景，它会**自动且强制性地暂停野人语**，瞬间切回 100% 通顺、礼貌且严谨的人类全量自然语言向你请求确认：

1. 检测到重大的**代码安全风险警告**（Security warnings）。
2. 涉及**不可逆或破坏性的本地 Shell 行为**（如执行 `rm -rf` 强删目录、强清数据库、修改核心权限配置）。
3. 包含严格要求前后顺序的**超多步骤复杂迁移指导**，防止碎片化语序导致人类产生误读。

***

## 🛠️ 第四部分：结合你的开源项目——实战拆解流水线

现在你的环境已经完美武装。建议你在进入 `claude` 界面后，通过以下“野人流水线”开启对当前 `D:\Project\claude\test` 目录的极速拆解：

1. **第一步**：输入 `/caveman full` 锁死完全野人模式。
2. **第二步**：丢入你的第一条宏观拆解 Prompt。
3. **第三步**：在它帮你梳理出技术栈和 `CLAUDE.md` 规范后，立刻输入 `/caveman-compress CLAUDE.md`，把这个文件的体积瞬间砍掉一半，为接下来的长周期二开埋下完美的“省钱伏笔”。
