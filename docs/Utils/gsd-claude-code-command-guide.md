# GSD（Get Shit Done）插件介绍与 Claude Code 常用命令指南

## 1. GSD 项目介绍

GSD（Get Shit Done）是一个面向 Claude Code 的 AI
软件开发工作流插件，目标是解决大型 AI 辅助开发过程中常见的问题：

-   长对话导致上下文丢失
-   AI 编程缺少明确规划
-   复杂项目容易偏离目标
-   多阶段任务难以持续推进
-   代码生成缺少工程化流程

GSD 采用「规格驱动开发（Spec-Driven
Development）」和「上下文工程（Context Engineering）」理念，将 AI
从简单代码生成工具提升为项目开发协作者。

仓库地址：

https://github.com/cmzt-coder/get-shit-done
https://github.com/gsd-build/get-shit-done

------------------------------------------------------------------------

## 2. GSD 核心理念

### 2.1 规划优先

GSD 不建议直接让 AI 编码，而是：

需求分析 ↓ 项目研究 ↓ 架构设计 ↓ 任务拆解 ↓ 逐步实现 ↓ 测试验证 ↓
经验沉淀

------------------------------------------------------------------------

### 2.2 状态驱动

GSD 会维护项目状态：

-   当前阶段
-   已完成任务
-   未完成任务
-   下一步计划
-   技术决策记录

避免 AI 在长周期开发中遗忘上下文。

------------------------------------------------------------------------

### 2.3 适合复杂项目

特别适合：

-   企业级应用
-   SaaS 系统
-   RAG 平台
-   Agent 系统
-   后端服务
-   全栈项目
-   自动化平台

------------------------------------------------------------------------

## 3. GSD 项目结构

初始化后通常会生成类似结构：

    .gsd/
    
    ├── PROJECT.md
    ├── REQUIREMENTS.md
    ├── ROADMAP.md
    ├── STATE.md
    └── research/

说明：

  文件              作用
----------------- ----------------
  PROJECT.md        项目目标和背景
  REQUIREMENTS.md   需求定义
  ROADMAP.md        开发路线规划
  STATE.md          当前状态记录
  research          技术调研资料

------------------------------------------------------------------------

## 4. Claude Code 中常用命令

### 4.1 项目初始化

#### 初始化项目

    /gsd new-project

创建 GSD 项目管理体系。

------------------------------------------------------------------------

#### 深度初始化

    /gsd new-project --deep

适用于：

-   大型系统
-   企业项目
-   AI 平台

会进行更完整的需求分析。

------------------------------------------------------------------------

## 5. 日常开发核心命令

### 查看项目状态

    /gsd status

查看：

-   当前阶段
-   完成情况
-   下一步任务

推荐每天开始开发前执行。

------------------------------------------------------------------------

### 自动推进开发

    /gsd auto

自动执行：

需求 →规划 →开发 →验证

适合：

明确的小型任务。

------------------------------------------------------------------------

### 手动推进模式

    /gsd

每一步等待确认。

适合：

-   架构设计
-   核心模块开发
-   生产系统建设

------------------------------------------------------------------------

## 6. 需求与架构设计

### 技术讨论

    /gsd discuss

用于：

-   技术选型
-   架构设计
-   方案比较

示例：

    /gsd discuss
    
    设计一个企业级RAG知识库系统

------------------------------------------------------------------------

### 创建里程碑

    /gsd new-milestone

用于拆分大型项目：

例如：

    Milestone 1 用户系统
    
    Milestone 2 知识库
    
    Milestone 3 AI Agent
    
    Milestone 4 运营后台

------------------------------------------------------------------------

## 7. AI 项目相关命令

### AI 集成设计

    /gsd ai-integration-phase

适用于：

-   RAG
-   Agent
-   LLM应用

关注：

-   模型选择
-   Prompt设计
-   数据流程
-   失败处理
-   评估体系

------------------------------------------------------------------------

### AI效果评估

    /gsd eval-review

检查：

-   AI功能完整性
-   输出质量
-   测试覆盖

RAG项目重点关注：

-   Recall
-   Precision
-   Faithfulness
-   Answer Quality

------------------------------------------------------------------------

## 8. 调试与维护

### 快速任务

    /gsd quick

适合：

-   修改Bug
-   小功能调整
-   简单优化

------------------------------------------------------------------------

### 调试

    /gsd debug

用于：

-   定位异常
-   分析失败流程

------------------------------------------------------------------------

### 深度分析

    /gsd forensics

适合：

-   AI执行混乱
-   状态异常
-   大型问题排查

------------------------------------------------------------------------

### 环境检查

    /gsd doctor

检查：

-   配置
-   环境
-   项目状态

------------------------------------------------------------------------

## 9. 知识沉淀

### 记录经验

    /gsd knowledge

保存：

-   架构决策
-   技术经验
-   避坑记录

例如：

    Milvus只保存chunk metadata，
    原始文档存储到对象存储。

------------------------------------------------------------------------

## 10. 推荐 AI 工程开发流程

针对 RAG / Agent 项目：

### 阶段1：项目初始化

    /gsd new-project --deep

### 阶段2：需求分析

    /gsd discuss

### 阶段3：架构规划

    /gsd new-milestone

### 阶段4：AI设计

    /gsd ai-integration-phase

### 阶段5：开发

大型任务：

    /gsd

明确任务：

    /gsd auto

### 阶段6：质量检查

    /gsd eval-review

### 阶段7：经验沉淀

    /gsd knowledge

------------------------------------------------------------------------

## 11. Claude Code + GSD 推荐配置理念

建议：

### 全局 CLAUDE.md

负责：

-   中文输出
-   编码规范
-   AI架构师定位
-   开发原则

### 项目 CLAUDE.md

负责：

-   项目技术栈
-   数据结构
-   架构约束
-   部署方式

### GSD

负责：

-   项目规划
-   任务拆解
-   状态管理
-   长周期执行

三者组合：

    CLAUDE.md
        +
    项目规范
        +
    GSD工作流
        =
    AI软件工程团队模式

------------------------------------------------------------------------

## 12. 总结

GSD 的核心价值不是增加几个命令，而是改变 AI 编程方式：

从：

    告诉AI写代码

变成：

    AI参与完整软件工程流程

对于 RAG、Agent、企业应用等复杂项目，GSD 可以显著提升：

-   项目可控性
-   架构一致性
-   长任务完成率
-   AI协作效率
