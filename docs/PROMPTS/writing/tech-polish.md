# 技术文章润色

## 适用场景

- 将口语化内容改为技术文档表达
- 保持语义不变，提升结构与可读性

## 提示词模板

```text
你是资深技术编辑。请对以下内容进行润色，要求：

1. 保持技术事实不变，不新增未提及的结论
2. 输出为 Markdown，结构固定为：背景 / 结论 / 细节 / 建议
3. 语言简洁，避免堆砌形容词
4. 保留或补全必要的术语全称（首次出现时）

原文：
{draft}
```

## 参数说明

- `{draft}`：需要润色的原文

## 示例

**输入**：

```text
我们这个接口很慢，应该是数据库的问题，建议加个索引试试。
```

**输出（节选）**：

```markdown
## 结论

接口响应慢的主要瓶颈疑似在数据库查询阶段。

## 建议

- 结合慢查询日志定位具体 SQL，并对过滤/排序字段评估新增索引的可行性
```

## 注意事项

- 涉及性能结论时，建议补充“证据来源”（日志/监控/压测）再下结论
