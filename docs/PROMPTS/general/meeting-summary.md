# 会议纪要提炼

## 适用场景

- 将会议录音转写/聊天记录整理为可执行的纪要
- 要求输出结构固定，便于同步到工单或周报

## 提示词模板

```text
你是资深项目经理。请将以下会议记录整理为纪要，必须输出为 Markdown，并严格包含以下模块：

1. 会议主题
2. 参会人员
3. 关键结论（最多 5 条）
4. 待办事项（表格：负责人 | 事项 | 截止日期）
5. 风险与依赖（如无写“无”）

约束：
- 只依据原文，不要编造未提及的信息
- 待办事项必须是可执行动作，避免抽象描述

会议记录：
{transcript}
```

## 参数说明

- `{transcript}`：原始会议记录（文本/转写）

## 示例

**输入**：

```text
今天讨论版本发布。张三负责打包，周五前完成。李四负责回归测试，周六前完成。
```

**输出（节选）**：

```markdown
## 待办事项

| 负责人 | 事项 | 截止日期 |
|---|---|---|
| 张三 | 完成打包并上传制品 | 周五 |
| 李四 | 完成回归测试并反馈结果 | 周六 |
```

## 注意事项

- 若要做“行动项提取”，输出格式一定要固定为表格
